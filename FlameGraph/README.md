# 📜 Инструкция по созданию и анализу FlameGraph в Linux

**FlameGraph** — это инструмент визуализации профилирования, который превращает тысячи строк логов `perf` в наглядную картинку. Он позволяет мгновенно найти «горячие точки» (Hotspots) в коде.

---

## 🛠 1. Подготовка (Установка)

Перед началом убедитесь, что в системе установлены инструменты профилирования и скрипты визуализации.

```bash
# 1. Обновляем списки пакетов
sudo apt update

# 2. Устанавливаем Git, Perl и инструменты perf для вашего ядра
sudo apt install git perl linux-tools-common linux-tools-$(uname -r) -y

# 3. Клонируем репозиторий со скриптами FlameGraph
git clone [https://github.com/brendangregg/FlameGraph](https://github.com/brendangregg/FlameGraph)
cd FlameGraph
```

## ⏺ 2. Сбор данных (Recording)
Мы используем утилиту perf для создания «снимков» стека вызовов (call stack) работающей системы.

Для конкретной команды (например, find):

```bash
sudo perf record -g find /usr > /dev/null
```
Для мониторинга всей системы (на 10 секунд):
```bash
sudo perf record -a -g sleep 10
```
## 🎨 3. Генерация FlameGraph (Rendering)
Теперь объединяем собранные данные в один интерактивный файл .svg.


###  Читаем данные -> Схлопываем стеки -> Создаем график
```bash
sudo perf script | ./stackcollapse-perf.pl | ./flamegraph.pl > profile.svg
```
## 🔍 Как правильно читать график
###  Основные принципы:
Ось Y (Вертикаль): Показывает глубину стека вызовов. Снизу вверх: от родительской функции (например, main) к дочерним.

Ось X (Горизонталь): Показывает долю ресурсов CPU. Чем шире блок функции, тем больше времени процессор находился в ней (или в функциях, которые она вызвала).

Порядок: Расположение по горизонтали обычно алфавитное, а не хронологическое.

###  Интерактивные возможности (откройте .svg в браузере):
Наведение (Hover): Показывает полное имя функции и точный процент её участия в нагрузке.

Клик (Zoom): Нажмите на блок, чтобы увеличить его на всю ширину экрана и изучить внутренние вызовы.

Поиск (Ctrl + F): Выделит все найденные функции фиолетовым цветом.
